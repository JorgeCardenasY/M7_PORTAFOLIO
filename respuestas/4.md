# Pregunta 4: Migraciones en Django


Para el desarrollo solicitado en Pregunta 4, se replicó el proyecto  Django `preg_2` en una nueva carpeta `preg_4`.

## Pasos realizados

1.  **Creación del nuevo proyecto:** Se creó una nueva carpeta `preg_4` y se copió el contenido del proyecto `preg_2`.

2.  **Creación del ambiente virtual:** Se creó un nuevo ambiente virtual en `preg_4/v_preg_4` y se instaló Django.

3.  **Modificación del modelo:** Se añadió un nuevo campo `stock` al modelo `Modelo1` en el archivo `preg_4/bi/models.py`.

    ```python
    class Modelo1(models.Model):
        name = models.CharField(max_length=100)
        description = models.TextField()
        price = models.DecimalField(max_digits=10, decimal_places=2)
        stock = models.PositiveIntegerField(default=0) # Nuevo campo
        created_at = models.DateTimeField(auto_now_add=True)
    ```

4.  **Creación de la migración:** Se ejecutó el comando `python manage.py makemigrations bi` para crear la migración correspondiente al cambio en el modelo.

5.  **Aplicación de la migración:** Se ejecutó el comando `python manage.py migrate` para aplicar la migración y actualizar el esquema de la base de datos.

## Verificación

Se puede verificar que la migración se ha creado en la carpeta `preg_4/bi/migrations/` y que la base de datos `preg_4/db.sqlite3` ha sido actualizada con el nuevo campo en la tabla `bi_modelo1`.
