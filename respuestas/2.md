
# Implementación de Modelos No Relacionados en Django

Para dar solución a la problemática de implementar una capa de modelo de acceso a datos con entidades no relacionadas, se ha creado modelos simples en Django. Este enfoque nos permite tener tablas completamente independientes en nuestra base de datos, lo cual es ideal para almacenar información que no necesita estar conectada entre sí.

## ¿Cómo lo hicimos?

La clave está en definir clases en nuestro archivo `models.py` que heredan de `django.db.models.Model`, pero sin establecer ningún campo de relación entre ellas (como `ForeignKey`, `OneToOneField` o `ManyToManyField`).

### Paso 1: Definir los Modelos

Dentro de una de nuestras aplicaciones de Django, abrimos el archivo `models.py` y definimos nuestras entidades. Por ejemplo, y sólo a modo de ejemplo, si estuviéramos construyendo una aplicación para una biblioteca, podríamos necesitar modelos para `Libro` y para `Lector`, pero sin que inicialmente estén relacionados. En concreto, en el archivo `models.py` se desarrolló del siguiente modo:

```python
from django.db import models

class Modelo1(models.Model):
    name = models.CharField(max_length=100)
    description = models.TextField()
    created_at = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return self.name

class Modelo2(models.Model):
    title = models.CharField(max_length=200)
    created_at = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return self.title
```

En este ejemplo, `Modelo1` y `Modelo2` son dos entidades completamente independientes. Un `Modelo1` no "sabe" a qué `Modelo2` pertenece, y un `Modelo2` no tiene una lista de `Modelo1`.

### Paso 2: Crear y Aplicar las Migraciones

Una vez que nuestros modelos están definidos, le pedimos a Django que prepare los "planos" para crear las tablas en la base de datos.

1.  **Crear las migraciones:**
    ```bash
    python manage.py makemigrations
    ```
    Este comando revisa los modelos, detecta los cambios (en este caso, la creación de `Producto` y `Categoria`) y genera un archivo de migración.

2.  **Aplicar las migraciones:**
    ```bash
    python manage.py migrate
    ```
    Este comando toma los archivos de migración y ejecuta el SQL necesario para crear las tablas en la base de datos (por ejemplo, `mi_app_modelo1` y `mi_app_modelo2`).

## Resultado en la Base de Datos

Después de ejecutar las migraciones, Django crea dos tablas en la base de datos:

1.  **Una tabla para `Modelo1`:** Con columnas para `id`, `name`, `description` y `created_at`.
2.  **Una tabla para `Modelo2`:** Con columnas para `id`, `title` y `created_at`.

Estas tablas son **independientes**. No hay `foreign keys` ni ninguna otra restricción que las vincule. Podemos agregar, modificar o eliminar registros en una tabla sin que afecte en lo más mínimo a la otra.
